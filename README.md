# Pathwise

**Don't know where to start? Start here.**

Pathwise is an AI-powered learning platform that turns any goal into a structured, traversable curriculum. Enter what you want to learn. Get a complete map of exactly what to study, in what order, and why — then work through it one concept at a time with AI-generated lessons tailored to your goal.

---

## The Problem

Most people who want to learn something serious — machine learning, systems design, a new programming language, a technical discipline — face the same wall: they don't know where to begin. Not because the information doesn't exist, but because the path through it doesn't.

Search returns articles. YouTube returns tutorials. Neither tells you what you actually need to know first, what builds on what, or how any of it connects to where you're trying to go.

Pathwise solves the map problem.

---

## How It Works

**1. Enter your goal.**
Type what you want to be able to do. Not a subject — a goal. "Build production-ready machine learning systems." "Understand how TCP/IP actually works." "Write a compiler from scratch."

**2. Get your learning path.**
Pathwise uses Claude Opus 4.7 to decompose your goal into a directed graph of atomic prerequisite concepts. Each concept is exactly one teachable idea, ordered so that every step builds on the last. The structure of the subject becomes visible at a glance.

**3. Learn, concept by concept.**
Open any node and Pathwise delivers a focused lesson: clear explanations, a concrete worked example, common misconceptions to watch for, and a self-guided practice prompt. Lessons are generated by Claude Sonnet 4.6, guided by a teaching plan embedded in the graph at generation time, and cached so they load instantly on every return visit.

**4. Mark complete. Move forward.**
Finishing a lesson unlocks the next. Your progress is saved. The path is always clear.

## Product

- **Skill Tree generation** — Claude Opus 4.7 produces a linear DAG of 8-18 atomic concepts grouped into sequential zones, with a structured teaching plan embedded in every node
- **AI-tutored lessons** — Claude Sonnet 4.6 delivers focused lessons per concept: explanatory sections, worked example, numbered misconceptions, optional interactive diagram, and a practice prompt
- **Persistent progress** — node states (current, available, completed) are tracked per user; lessons are cached permanently after first generation
- **Interactive graph** — zoomable, pannable canvas with chapter rails, node detail panels, and real-time progress indicators
- **Generation safety checks** — graph generation and save reject prohibited course requests and refusal-shaped placeholder roadmaps before they can render or persist

## Local Development

### Prerequisites

- Node.js 20+
- A Supabase project
- AWS Bedrock access (bearer token or IAM credentials)

### Setup

```bash
git clone https://github.com/your-org/ai-tutor
cd ai-tutor
npm install
```

Create `.env.local` with your Supabase and AWS credentials. The variables are listed below.

Apply database migrations via the Supabase dashboard SQL editor, running each file in `supabase/migrations/` in order.

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

The landing page exposes a `Join` waitlist button and a secondary `dev` button. `dev` opens `/sign-in`, which first shows a client-side development password gate before the Supabase sign-in form.

Lesson generation uses two caches. In normal operation, generated lessons are stored in `skill_nodes.generated_lesson`. During local development only, `/api/lessons/generate` also writes validated lesson payloads to `.pathwise-dev-cache/lessons.json` and reads that file before calling Bedrock. This keeps the currently saved learning path usable across reloads even while iterating on migrations or local database state. The cache directory is gitignored and is disabled in production. Set `PATHWISE_DISABLE_DEV_LESSON_CACHE=1` to bypass the local file cache.

## Design

Pathwise is built around a single conviction: the interface should disappear. The graph is the product. The lesson is a reading experience. Every element earns its place.

The visual system uses a restrained sage-and-ink palette (OKLCH design tokens, 12 light/dark theme variants), Bricolage Grotesque at heavy weights for structural hierarchy, JetBrains Mono for metadata and code, and a flat-by-default elevation model. Full light and dark mode. Brand assets (logo, favicon, app-icon) live in `/public/` as SVGs.

See [`DESIGN.md`](DESIGN.md) for the complete design system (including brand identity §7) and [`PRODUCT.md`](PRODUCT.md) for brand principles.
